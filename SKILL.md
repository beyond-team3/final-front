SKILL.md — MonSoon Frontend UI/UX Refactoring Guide

## 1) 목적
종자 영업 관리 시스템(ERP) 프론트엔드 UI/UX를 리팩토링한다.

핵심 목표:
- 역할별 사용자 경험 분리
- 정보 구조(IA) 재정렬
- 문서 파이프라인 가시성 강화
- 브랜드 일관성 및 가독성 개선

## 2) 프로젝트 컨텍스트
- 제품: B2B 종자 영업 ERP 시스템
- 사용자 역할:
  - 관리자(Admin)
  - 영업사원(Sales Rep)
  - 거래처(Client)

## 3) UX 방향성
- 톤: Clean + Professional + Subtle Nature
- 키워드: Seed, Growth, Cycle, Season, Trust
- 원칙:
  - 농업 도메인이지만 촌스럽지 않게 구성
  - 그린 계열은 포인트 용도로 제한
  - 재배 주기(Seasonality)와 문서 흐름을 UI에서 명확히 표현

## 4) 정보 구조(IA) 규칙
### 4.1 헤더(전역 기능)
다음 메뉴는 헤더로 이동하고 상단 고정한다.
- 일정
- 알림
- 설정

### 4.2 사이드바 구조
1. 대시보드
2. 영업 관리
   - 견적요청
   - 견적서
   - 계약서
   - 주문서
   - 명세서
   - 청구서
3. 상품 관리
   - 품종 관리
   - 재배적기(기존 위치에서 이동)
4. 고객 관리
   - 거래처
   - 사원 관리(관리자 전용)
5. 통계

## 5) 역할별 기능 요구사항
### 5.1 관리자 뷰
- 사원 상세에서 담당 거래처 리스트 노출
- 거래처 수 배지 및 KPI 강조
- 거래처 클릭 시 상세 페이지 이동
- 레이아웃:
  - 좌측: 사원 정보
  - 우측: 담당 거래처 카드 리스트

### 5.2 영업사원 뷰
- 청구서 취소 기능 추가
- 청구서 상태: `DRAFT`, `ISSUED`, `CANCELED`
- 취소 시 히스토리 기록 + 상태 배지 갱신

- 일정 필터에 재배적기 관련 항목 추가
- 필터 목록:
  - 문서 일정
  - 개인 일정
  - 재배적기
  - 수확 일정

- 재배적기 개선:
  - 품종별 수확 예정 시점 계산
  - 수확 월에 일정 자동 생성
  - 수확 월에 알림 자동 생성
  - "수확 시즌 알림" 카드 강조
  - 해당 거래처 바로 연락 CTA 제공

### 5.3 거래처 뷰
- 주문서 기능에 다음을 필수 반영:
  - 주문 취소 기능
  - 승인 단계 추가
- 주문 상태 흐름:
  - `DRAFT` → `REQUESTED` → `APPROVED` → `REJECTED` → `CANCELED`
- 취소 권한:
  - 거래처: `REQUESTED` 상태에서 취소 가능
  - 영업사원: `REQUESTED` 상태에서 취소 가능
  - `APPROVED` 이후 취소: 관리자 승인 필요

## 6) 디자인 시스템 규칙
### 6.1 컬러(60-30-10)
- Dominant(60):
  - Light: `#F6F8F7`
  - Dark: `#121212`
- Secondary(30):
  - Card background, Sidebar, Header surface
  - Light: `#FFFFFF`
  - Dark: `#1E1E1E`
- Accent(10):
  - Primary: `#1F5E3B`
  - Warning: `#E4A11B`
  - Danger: `#C44536`

제약:
- 순수 검정(`#000000`) 및 순수 흰색(`#FFFFFF`)의 텍스트/배경 직접 사용은 지양한다.

### 6.2 타이포그래피
- Heading: Pretendard SemiBold
- Body: Pretendard Regular
- Caption: 14px
- 본문 기본: 16px
- 줄간격: 1.6
- 시각적 위계: 최소 3단계 유지

### 6.3 간격 체계
- 8px Grid System
- 내부 패딩: 16px
- 카드 패딩: 24px
- 섹션 간격: 48px
- 대형 구분 간격: 64px 이상

## 7) 문서 상태 UI 규칙
- 모든 문서에 상태 배지 스타일을 통일한다.
- 상태 매핑:
  - `DRAFT`: Neutral
  - `REQUESTED`: Blue tone
  - `APPROVED`: Green tone
  - `REJECTED`: Red tone
  - `CANCELED`: Gray tone
- 배지 위치는 우측 상단 고정으로 통일한다.

## 8) 파이프라인 시각화 규칙
- 문서 흐름: 견적 → 계약 → 주문 → 명세 → 청구
- 상단 Progress Bar를 제공한다.
- 현재 단계는 강조한다.
- 완료 단계는 흐리게 처리한다.
- 취소 상태에서는 전체 흐름을 비활성화한다.

## 9) 접근성(필수)
- 텍스트 대비: 4.5:1 이상
- 버튼 대비: 3:1 이상
- 키보드 접근 가능해야 함
- 포커스 아웃라인: 2px 이상

## 10) 작업 체크리스트
- IA 변경이 헤더/사이드바에 반영되었는가
- 역할별 요구사항이 권한에 맞게 반영되었는가
- 문서 상태/파이프라인 규칙이 일관되게 적용되었는가
- 색상/타이포/간격 규칙을 위반하지 않았는가
- 접근성 기준(대비, 키보드, 포커스)을 만족하는가
